# rin-research-humanize

研究如何让模型在 QQ 群聊中的回复更像真实人类。

## 项目定位

当前的 LLM 群聊 bot 容易在几句话内被识破——不是因为答错，而是因为答得**太完整、太中立、每条都回、从不断句、没有情感节奏**。本项目系统性地研究一套可落地的方法，让 bot 回复在内容正确的前提下，带上人能感知到的"人味"。

这不是文艺项目，不研究"如何更有趣"——研究的是**哪些具体的机制让人一眼认出是人而不是 bot**，以及如何在工程上可控地模拟这些机制。

## 研究方向

### 1. 群聊节奏感 (Chat Rhythm)

群聊不是客服工单系统。真人不每条都回，回的时机、间隔、接话对象构成天然的轮次结构。研究如何：

- 根据消息重要性和社交关系决定"是否回"
- 根据群活跃程度调整回复间隔和长度
- 识别轮次边界——谁在接谁的话，什么时候话题断了

### 2. 风格指纹 (Style Fingerprint)

真人有稳定的语言指纹——口癖、常用连接词、断句习惯、标点偏好、表情频率。研究如何：

- 为 bot 建立一致的风格 profile（可配置多套）
- 在生成时将风格 profile 融入 prompt 而非事后润色
- 风格包含：称呼习惯、语气词频率、段落长度偏好、标点法则

### 3. 上下文接地 (Context Grounding)

真人的回复不只依赖上一句，还依赖群知识：这个群的缩写文化、共同的梗、谁和谁关系好、什么是敏感话题。研究如何：

- 从群聊历史中抽取群级常识（高频表情、黑话、禁忌）
- 感知社交关系链（谁经常一起出现、谁说话没人接）
- 在回复中自然调用这些群知识，而不是刻意展示

### 4. 不完美设计 (Imperfection by Design)

人类回复充满"瑕疵"——打错字不改、用空格代替标点、重复语气词、只回表情不说话、省略句。研究如何：

- 在生成中引入可控的非标准语法（不是随机加错，而是模式化的"真实瑕疵"）
- 控制瑕疵密度——太少不像人，太多像表演
- 区分场景（认真回复 vs 水群）调整不完美程度

### 5. 评估方法论 (Evaluation Framework)

"像不像人"无法单靠指标衡量。研究如何：

- 建立 A/B 盲测流程（真人 vs bot，群友投票）
- 设计可复现的测评集（覆盖不同群类型、话题、氛围）
- 定义客观辅助指标（回复率、首句多样性、表情使用分布）
- 平衡"像人"和"有用"——不要为了像人而变成废话 bot
