## 原因分析

### 可能原因 1：Bot 不在活跃群中

Bot QQ 号可能没有加入任何有实时消息的群聊，或加入的群在当前时段无人发言。OneBot 网关只转发它已收到的消息，不会主动生成。

### 可能原因 2：网关需额外订阅配置

部分 OneBot 实现（如 NapCat、Lagrange）需要显式开启消息上报开关。若未配置 `enable` 或 `report-self-message` 等选项，网关只接受连接但不转发消息。需要用户确认网关配置。

### 可能原因 3：Token 权限不足

`access_token` 能通过 WebSocket 握手鉴权（连接成功），但可能不具有消息推送的订阅权限。部分网关区分"连接鉴权"和"事件订阅"两个层级。

## 后续建议

1. **确认 Bot 所在群**：bot 号当前加了多少群？是否有活跃群？
2. **检查网关配置**：确认消息上报已开启（不同实现配置方式不同，需用户提供网关类型）
3. **缩短验证周期**：下次可先主动在群内发一条消息，验证消息是否能被网关转发到监听器
4. **增加日志**：监听器可考虑记录接收到的所有事件类型（包括非 message 事件），帮助判断网关是否在推送数据
